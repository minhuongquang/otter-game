# Documentation Rules

Documentation should explain decisions,

not duplicate source code.

---

# Architecture Documents

Include:

* Purpose
* Responsibilities
* Dependencies
* Data Flow
* Future Expansion

---

# Feature Documents

Every major feature should explain:

* Goal
* Scope
* Limitations
* Public API
* Related systems

---

# Decision Records

When architecture changes,

record:

* Problem
* Chosen solution
* Alternatives considered
* Why the decision was made

---

# Writing Style

Use Markdown.

Use short sections.

Use headings.

Avoid large paragraphs.

Prefer diagrams and lists where appropriate.

Documentation should be understandable months later.
